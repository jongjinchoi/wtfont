# Project Rules

## Communication
- 한국어 존댓말을 사용할 것. 반말 금지.

---

## 🚨 가장 중요한 규칙

### **1. 절대 추측 금지**
- **절대 추측하거나 가정하지 않기**
- **항상 실제 코드로 확인**
- **100% 코드 분석 필수**
- **존재하지 않는 정보를 절대 생성하거나 만들어내지 않기**

### **2. 빠른 해결 금지 - 근본적 해결만**
- **임시방편 해결책 적용 금지**
- **항상 근본 원인 먼저 이해**
- **적절하고 완전한 해결책 구현**
- **단기적이 아닌 장기적 사고**

---

## ❌ 하지 말아야 할 것들

### 1. 문서 업데이트
- 문서 업데이트 시 새 섹션 생성 금지
- 중복 콘텐츠 추가 금지
- 항상 기존 섹션 먼저 확인하고 새로 만들지 말고 업데이트할 것
- **중요: 항상 문서 전체를 먼저 읽기**
  - 업데이트 전에 문서 100% 읽기
  - 문서 내용에 대한 추측 금지
  - 모든 통계와 숫자를 실제 파일과 대조 확인
  - 모든 관련 섹션을 제대로 철저히 업데이트

### 2. 코드 변경
- 실제 코드 확인 없이 변경 금지
- 파일 위치나 내용 추측 금지 - 항상 Read 도구 먼저 사용
- 테스트/검증 없이 변경사항 적용되었다고 주장 금지
- 잘못된 파일에 스타일 추가 금지
- **중요: 파일 수정 전 항상 승인 받기**
  - 먼저 정확한 수정 내용 보여주기
  - AskUserQuestion 도구로 승인 요청
  - 사용자 승인 후에만 수정
- **중요: 파일 업데이트 시 항상 선택지로 물어보기**
  - 변경 방향이나 옵션이 여러 개일 때 선택지 제시
  - 사용자가 선택한 후에 진행

### 3. 일반적인 행동
- 추측하지 말고 항상 실제 파일로 확인
- 승인되지 않은 기능 무단 추가 금지
- 나중에 삭제될 무효한 작업으로 시간 낭비 금지
- 지적받은 실수 반복 금지
- **중요: 빌드/테스트는 "빌드해", "npm run build 해" 등 정확한 명령어 요청 시에만 실행**
  - `npm run build`, `npm test` 등 임의 실행 절대 금지
  - 구현 완료 후 자동으로 빌드/테스트 실행 금지
  - 확실하지 않으면 사용자에게 먼저 확인
  - **타입에러 체크 = `tsc --noEmit`** (빌드 아님)
  - **빌드 = `npm run build`** (명시적 요청 시에만)
- **중요: 분석 요구 시 절대 추측 금지 - 무조건 코드 100% 분석**
  - 언급된 모든 파일 읽기
  - 모든 주장을 실제 코드로 검증
  - 가정 금지, 추측 금지
- **중요: 외부 라이브러리 사용 시 무조건 공식 문서 100% 먼저 읽기**
  - 구현 전 공식 문서 완독
  - 라이브러리 기능이나 API에 대한 추측 금지
  - 모든 메소드와 속성을 공식 문서에서 확인
  - WebFetch/WebSearch로 최신 문서 읽기
- 서브에이전트(Task agent) 분석 결과는 반드시 직접 코드를 읽어 재검증할 것. 서브에이전트 결과를 그대로 전달하지 말 것.

### 4. 파일 관리
- 불필요한 새 파일 생성 금지
- 항상 새 파일 생성보다 기존 파일 수정 우선
- 명시적 요청 없이 문서 파일 생성 금지
